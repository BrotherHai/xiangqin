# 相亲网站设计文档

## 项目概览

亲友介绍型婚恋平台，管理员驱动的小范围相亲。移动端 + PC 自适应响应。

## 角色设计

| 角色 | 职责 |
|------|------|
| **管理员** | 审核资料、匹配推荐、管理性格测试题目、牵线流程管理 |
| **亲友/推荐人** | 代填征婚人资料、查看匹配推荐结果、确认/拒绝牵线 |
| **征婚人** | 查看自己的资料状态、做性格测试题 |

## 核心流程

1. **资料录入** — 亲友/管理员填写征婚人基本信息（照片、年龄、职业、择偶要求等）
2. **管理员审核** — 审核资料真实性和完整性
3. **性格测试** — 征婚人完成自定义性格测试（10-20 题，多维度评分）
4. **匹配推荐** — 管理员按条件 + 性格维度筛选，推荐匹配
5. **双方确认** — 双方亲友确认/拒绝牵线
6. **交换联系** — 双方同意后交换联系方式

## 功能模块

### 1. 资料管理
- 征婚人资料 CRUD（基本信息、照片、择偶要求）
- 资料状态：待审核 / 已通过 / 已拒绝
- 管理员可编辑所有资料

### 2. 性格测试系统
- 管理员管理测试题目（增删改）
- 题目标签维度（如外向/内向、理性/感性等）
- 征婚人答题后自动计算各维度得分
- 匹配时对比双方性格维度得分

### 3. 管理员匹配操作台
- 候选人列表展示
- 按基本条件（年龄、地区等）筛选
- 按性格维度筛选/排序
- 候选人与现有征婚人的匹配度展示

### 4. 牵线流程管理
- 管理员发起牵线推荐
- 双方亲友可查看匹配详情
- 确认/拒绝操作
- 牵线状态追踪（待确认 / 已通过 / 已拒绝 / 已交换）

## 技术架构

| 层面 | 选型 |
|------|------|
| 前端框架 | Next.js 14 (App Router) |
| UI 框架 | Tailwind CSS + Shadcn/ui |
| 数据库 | SQLite（开发）/ PostgreSQL（生产） |
| ORM | Prisma |
| 认证 | NextAuth.js（管理员） |
| 部署 | Vercel |

### 页面结构

```
/login                    — 管理员登录
/admin/dashboard          — 管理员仪表盘
/admin/profiles           — 资料管理（审核列表）
/admin/profiles/[id]      — 资料详情 / 编辑
/admin/tests              — 性格测试题目管理
/admin/match              — 匹配操作台
/admin/introductions      — 牵线列表
/test/[profileId]         — 征婚人答题页（公开无登录）
/profile/[id]             — 资料展示页（亲友查看）
```

### API 设计

```
POST   /api/auth/login          — 管理员登录
GET    /api/profiles            — 资料列表（分页+筛选）
POST   /api/profiles            — 创建资料
GET    /api/profiles/[id]       — 资料详情
PUT    /api/profiles/[id]       — 更新资料
PUT    /api/profiles/[id]/status— 审核状态变更

GET    /api/tests               — 测试题列表
POST   /api/tests               — 新增题目
PUT    /api/tests/[id]          — 编辑题目
DELETE /api/tests/[id]          — 删除题目
POST   /api/tests/[id]/submit   — 征婚人提交答案

GET    /api/match               — 匹配推荐列表
POST   /api/introductions       — 发起牵线
GET    /api/introductions       — 牵线列表
PUT    /api/introductions/[id]  — 确认/拒绝牵线
```

### 数据库模型（主要）

- **Profile** — 征婚人资料（姓名、性别、年龄、地区、职业、照片、择偶要求、状态等）
- **TestQuestion** — 性格测试题目（题目内容、维度标签、选项）
- **TestAnswer** — 征婚人答题记录
- **Match** — 匹配推荐（推荐人、被推荐人、状态）
- **Introduction** — 牵线记录（双方 profile、管理员、状态）
- **Admin** — 管理员账号

## 非功能需求

- 响应式设计：Tailwind CSS 断点适配移动端 / 平板 / PC
- 权限控制：管理员页面需登录，公开页面无需认证
- 数据安全：敏感信息仅管理员可见，牵线确认后才交换联系方式
